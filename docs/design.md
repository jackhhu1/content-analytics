## Problem Statement

Content creators (1K–50K followers) need a "Signal-to-Noise" filter for their niche. Manual scrolling only surfaces absolute popularity (big accounts getting big views). The product identifies **Relative Velocity**: posts from small-to-mid accounts that have broken out of their following size.

The product replaces:
1. **Vanity Benchmarking:** Comparing yourself to 1M+ follower accounts.
2. **High-Cost Scraping:** Storing media and running expensive Vision AI on content that doesn't need it.

## The "Alpha" Metric: Viral Coefficient (VC)

The core value proposition is finding content where:
$$VC = \\frac{\\text{Views}}{\\text{Follower Count}}$$

An **Overperformer** is defined as a post where the $VC$ is $\\geq 2.0$ (or $1.5x$ the account's median $VC$ for the last 30 days). This identifies "format-market fit" regardless of the creator's existing audience size.

## Updated Approaches

### Approach: "Metadata-Only Signal" (Recommended)
Focus on the ranking and the math. Let Instagram handle the media.
- **Pros:** 90% lower COGS, zero storage management, high-speed discovery.
- **Cons:** No permanent swipe file of visual assets (links only).

## Recommended Architecture

### 1. Niche Intelligence Scraper (Weekly Cron)
- **Input:** 5-10 Instagram handles per user.
- **Process:** Apify Instagram Scraper fetches only `metadata` (post URL, views, likes, comments, timestamp) and the account's current `follower_count`.
- **Constraint:** No image/video downloads. Only store the `thumbnail_url` for temporary UI display (proxied, not stored).

### 2. Ranking Engine (The Filter)
- **Baseline:** Calculate the median $VC$ for each tracked account.
- **Scoring:** Identify "Outliers" ($Post VC \geq 2.0$ OR $Post VC > 2x Account Median VC$). This global floor ensures consistently viral creators aren't penalized by their own high baseline.
- **Sorting:** The main feed shows the top 20 outliers across the user's entire niche, regardless of account size.

### 3. Lean UI (The Dashboard)
- **Feed:** A "Leaderboard" style list showing:
    - Account Name & Follower Count.
    - The "Outlier Multiplier" (e.g., "3.5x More Views than Followers").
    - Post Caption (first 2 lines).
    - "Open in Instagram" CTA.
- **Playbook:** A simple "Save to Playbook" button that stores the URL and a `user_notes` text field for the creator to draft their "twist."

## Revised Data Model

- `global_accounts`: `id`, `handle`, `current_follower_count`, `last_scraped_at`. (Global pool prevents redundant scraping costs if multiple users track the same niche).
- `user_subscriptions`: `user_id`, `account_id`. (Junction table for feed visibility via RLS).
- `posts`: `id`, `account_id`, `url`, `view_count`, `follower_count_at_scrape`, `viral_coefficient`, `is_outlier`, `scraped_at`, `thumbnail_url`.
  *Note: Webhook uses `ON CONFLICT (account_id, url) DO UPDATE SET view_count=EXCLUDED.view_count, viral_coefficient=EXCLUDED.viral_coefficient, is_outlier=EXCLUDED.is_outlier` to ensure metrics aren't frozen on the first scrape.*
- `playbook`: `id`, `user_id`, `post_id`, `hook_draft`, `format_notes`.

## Pre-Build Validation (The 24-Hour Test)
Instead of building the scraper, manually find 10 overperformers in a niche using `Views / Followers`. Put them in a Google Sheet. Show it to 3 creators. If they find the "Signal" (the high VC posts) more valuable than their own "Saved" folder, build the automated version.

## Key Constraints
- **Link Expiry:** Instagram CDN URLs expire and block hotlinking (403s) if embedded directly. Use a lightweight Next.js passthrough route (e.g., `/api/image-proxy?url=...`) to stream images server-side, masking the origin and resolving 403s without incurring Supabase storage costs.
- **Follower Accuracy:** Must scrape follower count *at the time of the post analysis* to ensure the Viral Coefficient isn't skewed by account growth.
